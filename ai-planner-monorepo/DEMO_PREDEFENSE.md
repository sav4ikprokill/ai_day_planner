# Demo script for pre-defense

## Цель демонстрации

Показать, что AI Day Planner — это не просто CRUD-интерфейс, а дипломный MVP с:
- вводом задач естественным языком;
- голосовым вводом;
- AI/fallback parsing;
- хранением в PostgreSQL;
- фоновыми jobs;
- отдельным worker;
- сценарием привычек.

## Что запустить заранее

```bash
docker compose up --build -d postgres api worker web-platform
docker compose ps -a
```

Дополнительно проверить:
- `http://127.0.0.1:5173`
- `http://127.0.0.1:8000/docs`
- голосовой ввод в Chrome или Edge

## Основной сценарий показа

1. Коротко назвать тему диплома.
2. Сказать, что core MVP состоит из web demo, FastAPI backend, PostgreSQL и Go worker.
3. Сразу отделить `experimental/reports-java` и `experimental/scheduler-cpp` от core.
4. Открыть web demo.
5. Создать задачу текстом:
   - `купить продукты завтра в 18:00`
6. Показать, что задача появилась в списке.
7. Изменить статус задачи.
8. Показать голосовой ввод:
   - нажать `Голосовой ввод`
   - произнести `добавь тренировку завтра в 19:00`
   - показать, что текст подставился в поле
   - вручную нажать создание задачи
9. Показать, что голосовой ввод не делает автосохранение.
10. Создать привычку, например:
   - `спорт вечером`
11. Открыть Swagger `/docs`.
12. Сказать, что backend сначала может использовать Gemini, а при недоступности внешнего AI работает через fallback parser.
13. При необходимости показать таблицу `jobs` или объяснить, что фоновые действия идут через PostgreSQL queue и Go worker.

## Если времени мало

Показывать только:
- текстовое создание задачи;
- голосовой ввод;
- список задач;
- смену статуса;
- создание привычки;
- Swagger `/docs`.

## Что говорить про голосовой ввод

- Голосовой ввод реализован на клиентской стороне через Web Speech API: браузер распознаёт речь и передаёт backend уже текстовую команду.
- Аудио на сервер не отправляется и не хранится.
- Пользователь видит распознанный текст и сам подтверждает создание задачи.
- Поддержка зависит от браузера и доступа к микрофону.

## Что говорить про качество

- Качество parser оценивалось на небольшом ручном evaluation-наборе из 30 фраз.
- Понятность сценария проверялась через маленький qualitative usability test на 3–5 пользователях.
- Это не production-grade benchmark, а честная MVP-оценка.

## Если что-то пошло не так

### Если не работает Gemini

- сказать, что в MVP предусмотрен fallback parser;
- продолжить demo через базовый сценарий создания задачи.

### Если не работает голосовой ввод

- сразу перейти на текстовый ввод;
- честно сказать, что это ограничение браузерного Web Speech API или прав на микрофон;
- не тратить на это много времени.

### Если не нужен experimental блок

- не открывать Java/C++;
- просто сказать, что это экспериментальные расширения архитектуры.

## Финальная формулировка

AI Day Planner — это дипломный MVP, где пользователь может создать задачу текстом или голосом, система преобразует её в структурированную задачу, сохраняет в PostgreSQL и обрабатывает фоновые действия через jobs queue и worker. Ключевая идея проекта — удобный естественный ввод плюс устойчивый core flow даже при ограничениях внешнего AI.
